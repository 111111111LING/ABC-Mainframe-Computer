## 二、5A A5 帧协议 (TCP)

### 2.1 请求帧 (PC → CH32V317)

```
 字节偏移    值              说明
 ──────────────────────────────────────────────────
  0          0x5A            帧头1
  1          0xA5            帧头2
  2          CMD             命令码
  3          DLEN_L          数据长度 (低字节)
  4          DLEN_H          数据长度 (高字节)
  5..        DATA            DLEN 字节的有效载荷
```

> DLEN = `DLEN_L | (DLEN_H << 8)`, little-endian 16位无符号整数

### 2.2 响应帧 (CH32V317 → PC)

```
 字节偏移    值              说明
 ──────────────────────────────────────────────────
  0          0x5A            帧头1
  1          0xA5            帧头2
  2          CMD | 0x80      响应命令码 (原CMD最高位置1)
  3          STATUS_LEN_L    状态数据长度 (低字节)
  4          STATUS_LEN_H    状态数据长度 (高字节)
  5          STATUS          状态码 (0=成功, 0xFF=失败)
```

> 对于目前使用的 CMD 0x01/0x02/0x05 的 ACK，`STATUS_LEN=1`，只有1字节状态码。

## 四、数据包详细结构

### 4.1 CMD 0x01 — 设备身份 (129 字节)

```
 偏移  长度  字段        类型        说明
 ─────────────────────────────────────────────────
  0    32    ProductID   bytes       产品ID, 不足补0x00
 32    32    DeviceName  bytes       设备名称, 不足补0x00
 64    64    SecKey      bytes       安全密钥, 不足补0x00
 128    1    Bind        uint8       绑定标志 (0/1)
```

**Rust打包代码 (protocol.rs):**
```rust
pub fn pack_device_identity(config: &DeviceConfig) -> Vec<u8> {
    let mut data = Vec::with_capacity(129);
    let mut buf = [0u8; 32];
    // ProductID (32 bytes)
    let pid = config.product_id.as_bytes();
    let len = pid.len().min(32);
    buf[..len].copy_from_slice(&pid[..len]);
    data.extend_from_slice(&buf);
    // DeviceName (32 bytes)
    buf.fill(0);
    let dn = config.device_name.as_bytes();
    let len = dn.len().min(32);
    buf[..len].copy_from_slice(&dn[..len]);
    data.extend_from_slice(&buf);
    // SecKey (64 bytes)
    let mut buf64 = [0u8; 64];
    let sk = config.sec_key.as_bytes();
    let len = sk.len().min(64);
    buf64[..len].copy_from_slice(&sk[..len]);
    data.extend_from_slice(&buf64);
    // Bind
    data.push(config.bind);
    data
}
```

**Python 示例 (完整打包):**
```python
def pack_device_identity(product_id, device_name, sec_key, bind):
    data = bytearray(129)
    data[0:32] = product_id.encode().ljust(32, b'\x00')[:32]
    data[32:64] = device_name.encode().ljust(32, b'\x00')[:32]
    data[64:128] = sec_key.encode().ljust(64, b'\x00')[:64]
    data[128] = bind
    return bytes(data)
```

### 4.2 CMD 0x02 — 网络配置 (31 字节)

```
 偏移  长度  字段           类型        说明
 ──────────────────────────────────────────────────────
  0     1    DHCP           uint8       DHCP启用标志 (0/1)
  1     4    IP             uint8[4]    静态IP地址
  5     4    Subnet Mask    uint8[4]    子网掩码
  9     4    Gateway        uint8[4]    网关
 13     6    MAC            uint8[6]    MAC地址
 19     4    MQTT Server IP uint8[4]    MQTT服务器地址
 23     2    MQTT Port      uint16 LE   MQTT端口
 25     4    NTP Server IP  uint8[4]    NTP服务器地址
 29     2    NTP Port       uint16 LE   NTP端口
```

**Rust打包代码 (protocol.rs):**
```rust
pub fn pack_network_config(config: &NetworkConfig) -> Vec<u8> {
    let mut data = Vec::with_capacity(31);
    data.push(config.lan_dhcp as u8);
    data.extend_from_slice(&parse_ip(&config.lan_ip));
    data.extend_from_slice(&parse_ip(&config.lan_mask));
    data.extend_from_slice(&parse_ip(&config.lan_gateway));
    data.extend_from_slice(&parse_mac(&config.mac_addr));
    data.extend_from_slice(&parse_ip(&config.mqtt_server_ip));
    data.extend_from_slice(&config.mqtt_server_port.to_le_bytes());
    data.extend_from_slice(&parse_ip(&config.ntp_server));
    data.extend_from_slice(&config.ntp_port.to_le_bytes());
    data
}
```

**Python 示例:**
```python
import struct

def pack_network_config(dhcp, ip, mask, gw, mac, mqtt_ip, mqtt_port, ntp_ip, ntp_port):
    data = bytearray(31)
    data[0] = 1 if dhcp else 0
    data[1:5] = [int(x) for x in ip.split('.')]
    data[5:9] = [int(x) for x in mask.split('.')]
    data[9:13] = [int(x) for x in gw.split('.')]
    mac_hex = mac.replace(':', '')
    data[13:19] = bytes.fromhex(mac_hex)
    data[19:23] = [int(x) for x in mqtt_ip.split('.')]
    struct.pack_into('<H', data, 23, mqtt_port)
    data[25:29] = [int(x) for x in ntp_ip.split('.')]
    struct.pack_into('<H', data, 29, ntp_port)
    return bytes(data)
```

### 4.3 CMD 0x05 — IAP 317 升级 (0 字节 DATA)

**流程:**
```
PC → 317:  5A A5 05 00 00
317 → PC:  5A A5 85 01 00 STATUS    (STATUS=0 成功)
PC → 317:  [原始 .bin.bin 数据]
317: 校验头 → 校验固件 → 写入flash → 写IMAGE_FLAG_UPDATE → 重启
```

**IAP 头格式 (512字节，固定):**
```
 偏移  长度  字段           类型        说明
 ─────────────────────────────────────────────────
  0     8    Magic          bytes       "WCHNET\0\0"
  8     4    TotalLength    uint32 LE   总长度 = 512 + 固件大小
 12     4    Checksum       uint32 LE   固件校验和 (不含头)
 16   496   Padding         bytes       填充0x00
512     N    FirmwareData   bytes       固件二进制数据
```

**校验和算法:**
```python
def calculate_checksum(data):
    total = 0
    for i in range(0, len(data), 4):
        chunk = data[i:i+4]
        val = int.from_bytes(chunk, 'little', signed=False)
        total = (total + val) & 0xFFFFFFFF
    return total
```

**Rust 生成代码 (firmware.rs):**
```rust
fn calculate_checksum(data: &[u8]) -> u32 {
    let mut sum: u32 = 0;
    for chunk in data.chunks_exact(4) {
        sum = sum.wrapping_add(u32::from_le_bytes(chunk.try_into().unwrap()));
    }
    let remainder = data.chunks_exact(4).remainder();
    if !remainder.is_empty() {
        let mut padded = [0u8; 4];
        padded[..remainder.len()].copy_from_slice(remainder);
        sum = sum.wrapping_add(u32::from_le_bytes(padded));
    }
    sum
}

// 输出: IAP_HEADER(512B) + FIRMWARE
// 写入文件: DNS.bin.bin
```

### 5.3 317 固件升级

```
上位机 (已连接)                      CH32V317 (SELECT模式)
     │                                    │
     │ 5A A5 05 00 00                     │
     │───────────────────────────────────→ │ CMD_IAP_317

     │ [原始 .bin.bin 数据流]              │
     │───────────────────────────────────→ │ saveUpdatedFile()

     │                                     │ - 校验数据
     │                                     │ - 校验通过 → 写IMAGE_FLAG_UPDATE

     │                                     │ - bootloader读取IMAGE_FLAG
     │                                     │ → 跳转 APP (新固件运行)
```

## 六、Flash 布局 (IAP_Task.h)

```
0x08000000  ┌──────────────────────┐
            │  BIM (bootloader)    │  40KB
0x0800A000  ├──────────────────────┤
            │  USER (APP区域)       │  152KB
0x08030000  ├──────────────────────┤
            │  BACKUP (备份区域)    │  152KB
0x08055F00  ├──────────────────────┤
            │  升级标志区 (1页256B) │
            │  0x00: IMAGE_FLAG    │  "SELE"=SELECT / "WCH "=UPDATE
            │  0x04: CFG1标志      │  "CFG1" (4B)
            │  0x08: CFG2标志      │  "CFG2" (4B)
            │  0x20: 设备身份配置   │  129B
            │  0xB0: 网络配置       │  31B
            └──────────────────────┘
```

**标志定义:**
| 值 | 含义 |
|----|------|
| `0x53454C45` ("SELE") | SELECT模式 — 等待上位机配置 |
| `0x57434820` ("WCH ") | UPDATE模式 — 拷贝BACKUP→USER后跳APP |
| 其他 | 正常启动 — 直接跳USER区域APP |

**APP启动后读取配置 (APP.c ReadFlashConfig):**
```c
void ReadFlashConfig(void) {
    // 从 0x08055F20 读取129B 设备身份
    // 从 0x08055FB0 读取31B 网络配置
    // 解析IP/Mask/GW/MAC/MQTT/NTP...
    net[25..28] = NTP_ServerIP;
    net[29..30] = NTP_port;
}
```
